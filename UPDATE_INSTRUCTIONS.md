# تعليمات إضافة تحديث جديد

## الخطوات:

### 1. إضافة معلومات التحديث في Firebase

افتح Firebase Console وانتقل إلى Realtime Database، ثم أضف هذا المسار:

```
appVersion/
  ├─ version: "1.0.1"
  ├─ downloadUrl: "رابط التنزيل المباشر للملف .exe"
  └─ notes: "- إضافة ميزة التحديث التلقائي\n- تحسينات في الأداء\n- إصلاح بعض الأخطاء"
```

### 2. مثال على البيانات في Firebase:

```json
{
  "appVersion": {
    "version": "1.0.1",
    "downloadUrl": "https://drive.google.com/uc?export=download&id=YOUR_FILE_ID",
    "notes": "- إضافة حقل اللقب للعملاء والمشرفين\n- تحسين نظام الطباعة\n- إضافة زر واتساب للمشرفين والمديرين"
  }
}
```

### 3. رفع ملف التحديث:

يمكنك رفع ملف .exe الجديد على:
- **Google Drive** (مشاركة عامة)
- **Firebase Storage**
- **أي خادم آخر**

### 4. الحصول على رابط مباشر من Google Drive:

1. ارفع الملف على Google Drive
2. اضغط بزر الماوس الأيمن > مشاركة > أي شخص لديه الرابط
3. انسخ ID الملف من الرابط
4. استخدم: `https://drive.google.com/uc?export=download&id=FILE_ID`

### 5. تحديث رقم الإصدار في package.json:

قبل بناء التطبيق، غيّر الإصدار في `package.json`:

```json
{
  "version": "1.0.1"
}
```

### 6. بناء التطبيق:

```bash
npm run build
```

## ملاحظات:

- التطبيق يتحقق من التحديثات كل 6 ساعات تلقائياً
- المستخدم يمكنه التحقق يدوياً من قائمة "مساعدة" > "التحقق من التحديثات"
- يجب أن يكون رقم الإصدار في Firebase أكبر من الإصدار الحالي
